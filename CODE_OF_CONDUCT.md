# Code of Conduct

## Our Pledge

We want Cabinet to be a calm, constructive place to collaborate. Contributors, maintainers, and participants are expected to treat each other with respect and to help keep discussions clear, practical, and welcoming.

## Expected Behavior

- Be respectful in disagreement.
- Assume good intent, but be clear about problems.
- Give actionable feedback.
- Keep discussion focused on the work.
- Respect different levels of experience and different working styles.

## Unacceptable Behavior

- Harassment, intimidation, or personal attacks.
- Discriminatory or hateful language.
- Deliberate disruption, trolling, or bad-faith argument.
- Publishing private information without permission.
- Any other conduct that would make collaboration unsafe or needlessly hostile.

## Scope

This code of conduct applies in the repository, issue discussions, pull requests, and any other project spaces used to collaborate on Cabinet.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, or contributions that violate this code of conduct. Repeated or severe violations may result in temporary or permanent exclusion from the project.

## Reporting

If you encounter unacceptable behavior, report it to the maintainers through the channel provided by the repository owner. Reports should include enough context to evaluate the issue fairly.

## Maintainer Standard

Maintainers are expected to enforce this code of conduct consistently and to explain moderation decisions when doing so is safe and appropriate.
